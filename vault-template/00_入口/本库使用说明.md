# 本库使用说明

这是 `你的 Obsidian Vault` 的入口页。新版系统的核心是：**围绕执行和项目展开**。捕获、分类、仪表盘、AI 记忆、资料和复盘都只服务于项目推进和今日执行。

## 每天默认只打开这两个

- `10_Daily/当天日期.md`
- [[../70_Dashboards/项目推进台]]

有新捕获需要处理时，再打开：

- [[../70_Dashboards/Inbox处理台]]

如果要看全局任务，再打开：

- [[../70_Dashboards/任务总览]]

## 日常流程

1. 捕获：任何内容先进 [[../00_Inbox/收件箱]]。
2. 判断：能今天做完就进 Daily；需要分步就进 Project；无价值就丢弃。
3. 项目推进：打开 [[../70_Dashboards/项目推进台]]，选择 1-3 个下一步。
4. 路由：只有 Inbox 内容影响执行或需要清理时，打开 [[../70_Dashboards/Inbox处理台]]。
5. 执行：只看当天 Daily 和当前 Project 的下一步。
6. 复盘：晚上让 Agent 生成复盘建议，确认后沉淀。

## 文件夹用途

- `00_Inbox`：临时收集，等待 Agent 路由。
- `10_Daily`：每日执行入口。
- `20_Goals`：规划、指标、阶段计划，用于项目决策。
- `30_Projects`：有完成标准的项目，是分步执行主线。
- `40_Areas`：长期责任领域，只放原则、维护清单、SOP 和项目索引。
- `50_Resources`：课程、文章、工具流程、图片、外部资料，用于支撑项目。
- `60_Reviews`：复盘和异常处理，用于改进项目推进。
- `70_Dashboards`：项目、任务、目标和数据展示，用于决策，不用于录入。
- `80_AI_Memory`：给 AI 长期使用的稳定背景。
- `90_Templates`：模板。
- `99_Archive`：完成或过期内容归档。
