# 项目推进台

这是新版系统的默认执行仪表盘。它不是录入区，而是帮助你决定“今天推进哪个项目、下一步是什么、哪里卡住了”。

## 什么时候打开

- 每天早上确定今日 1-3 件任务。
- 白天不知道下一步该做什么。
- 晚上复盘未完成任务。
- 周日检查所有项目是否仍然有下一步。

## 使用原则

先看项目，再看任务。

```text
Project → 下一步 → Daily
```

如果一条任务不属于任何项目，也不是今天能完成的小事，就先让 Agent 判断是否需要新建项目。

## 今日项目任务

```tasks
not done
path includes 30_Projects
due before tomorrow
sort by priority
sort by due
```

## 项目下一步

这里显示所有项目区未完成任务。每天只从中挑 1-3 个进入 Daily。

```tasks
not done
path includes 30_Projects
sort by priority
sort by due
sort by path
```

## 阻塞项目

任务描述里包含 `#blocked` 会出现在这里。

```tasks
not done
path includes 30_Projects
description includes #blocked
sort by path
```

## 本周到期的项目任务

```tasks
not done
path includes 30_Projects
due after yesterday
due before in 8 days
sort by due
sort by priority
```

## 活跃项目索引

如果 Dataview 插件已启用，并且项目文件包含 `type: project` 或 `status: active`，这里会聚合项目状态。

```dataview
TABLE status AS 状态, area AS 领域, goal AS 目标, next_action AS 下一步, due AS 截止
FROM "30_Projects"
WHERE status = "active"
SORT due ASC
```

## 给 Agent 的项目推进提示词

```text
请按执行中心原则检查我的项目推进状态：
1. 哪些项目没有明确下一步？
2. 哪些项目应该进入今天 Daily？
3. 哪些任务太大，需要拆小？
4. 哪些项目阻塞，需要我做决策？
5. 不要处理 Inbox，除非它影响当前项目推进。
先给建议表，不要直接改文件。
```
